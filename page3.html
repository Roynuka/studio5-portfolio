Start of a "wonderful" Journey

Router Finding
Before the start of one our studio 5/project 2 classes. Rob grabbed Scott and I and tasked us with finding 2 of the routers that ran the latest version of the router along with Ipv6. There were a total of 6 routers we had to go through to find them. At first we thought that they all had the same version running but after careful examination I noticed that in fact there was actually two of them that did. 
Load DB2 Users

At first I wasn't assigned to this task but I was present at the process of the task being completed. Scott, Yan, and Regan we're all working together to get it working before I joined in on the process. At first we couldn't figure out why the scripts weren't fully working as the output was missing the 'full name'. After vigorous testing we still couldn't fix the issue. Then Rob mentioned something to do with the file being a Windows file being run on a Unix server. Something we never would have figured out.  So basically we had to covert the file to a 'Unix' file. This is done by opening the script on notepad ++. From there go to edit>EOL Conversion>Unix. From there copy the file over through WinSCP to the DB2 server and run the script again. 

Load DB3 Users
A task was posted by Rob a few weeks back on loading DB2 and DB3 users onto their servers. Initially I was going to task this to myself as I added the task on the 'things to do' tab. But others in the group assigned them and also I already had tasks to do. Couple of weeks later I decided to jump on as I realized my portfolio was looking a little thin. Together with Yan we worked on loading users onto the DB3 server and adding them to the docker group. Already solving the issue we had in loading the DB2 users on the DB2 server it was just a matter of resusing the same script with a different list of users in a txt file and a few extra lines of code.
